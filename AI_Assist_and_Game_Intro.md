# 网易入职个人作业：作品介绍与 AI 辅助记录（MyRPGGame）

## 作品概述（一句话）
这是一个基于 **Unity 2D + Mirror** 的小型联机对抗/守矿玩法：玩家通过**击败野怪、拾取金块并投递到金矿**来推进胜利；同时要防止野怪偷矿与对方干扰。

---

## 游戏玩法说明（演示时可口述）

### 核心目标与胜负
- **胜利条件**：在回合计时结束前，将金矿当前金量补满到上限（`currentGold >= maxGold`）显示 **Victory**。
- **失败条件**：时间耗尽仍未补满 / 金矿被清空 /（多人模式下）所有玩家死亡后显示 **Game Over**。

### 玩家操作循环
- **移动**：键盘 WASD/方向键移动，带死区防止漂移。
- **攻击**：近战连击（基于动画状态与命中帧事件）；伤害结算由服务器权威进行。
- **拾取与投递金块**
  - 地面存在 `GoldPickup` 金块，可触碰拾取（服务端最终校验与扣除）。
  - 进入金矿触发范围后可按键投递，把背包金块存入金矿。
  - 支持按键丢弃金块（用于走位、资源博弈）。
- **技能系统**：通过 SkillSO 配置按键与冷却，示例包括：
  - **冲刺（Dash）**
  - **环绕火焰（Fire Orbit）**
  - **治疗（Heal）**

### 野怪机制与资源博弈
- 野怪具备基础 AI：索敌、追击、攻击（部分敌人带远程技能）。
- 野怪可对金矿造成压力（偷矿/带走金块等），迫使玩家在“打怪/采集/投递/防守”之间做权衡。
- **掉落机制**
  - **携带金块的野怪死亡**会掉落金块，双方都可见、可拾取。
  - **玩家携带金块死亡**会掉落其身上携带的全部金块，双方都可见、可拾取。

### 死亡状态的一致性体验
玩家死亡后进入“死亡锁定”：
- 不能移动、不能攻击、不能放技能
- 不能捡金、不能投递
- 关闭碰撞体积与刚体模拟
- 敌人/弹丸不再将其视为有效目标（避免“尸体继续被打”）

## 技术实现要点（师傅更关心的部分）

### 1) 联机框架：Mirror（权威服务器 + 同步）
- 关键交互采用 **服务器权威**：
  - 拾取、投递、伤害、技能生成、掉落生成、胜负结算
- 使用 Mirror 的同步机制：
  - **`[SyncVar]`**：生命、胜负状态、携带状态等
  - **`[Command]`**：客户端发起请求，服务器执行权威逻辑（如拾取/施法）
  - **`[ClientRpc]` / `[TargetRpc]`**：广播或仅对拥有者同步音效/表现（避免双端重复播放）

### 2) 2D 物理与动画事件驱动的战斗
- 使用 `Collider2D` Trigger 进行拾取/投递触发。
- 近战命中使用 **动画事件**回调，在服务器进行 `OverlapCircleAll` 判定与伤害结算，减少表现/结算分离导致的不一致。

### 3) 技能系统：ScriptableObject（可扩展）
- 通过 `SkillSO` 抽象技能配置（按键、冷却），用 `DashSkillSO / FireSkillSO / HealSkillSO` 等派生实现逻辑。
- 技能由 `PlayerSkills.CmdActivateSkill` 在服务器触发，确保多人一致。

### 4) 同步一致性：掉落/携带/死亡/结算
该作业重点解决了多人同步中的常见“Host 正常但 Client 异常”问题：
- **携带金块掉落在 Client 不显示/不可拾取**：通过网络生成策略与携带状态判定兜底，保证双方一致可见可拾取。
- **玩家死亡禁用交互**：死亡后统一禁用移动/战斗/技能/拾取投递，并关闭碰撞，避免继续被索敌或触发交互。
- **胜负 UI 在 Host/Client 不一致**：修复 SyncVar hook 触发顺序导致的标题显示错误。

## 启动与演示流程（展示给师傅时的操作步骤）

### 单人演示（本机）
1. 点击 **单人开始**（Host 模式，直接开局）
2. 展示移动、攻击、技能、拾取/投递、野怪行为、胜利/失败结算

### 双人演示（两台机器或同机两开）
1. A 点击 **双人开始**
2. B 输入 A 的局域网地址与端口，点击 **双人开始**（或直接作为 Client 连接）
3. 对方加入后显示“对方已准备好”，双方进入对局

> 备注：等待对方加入期间会锁定本地玩家操作，避免“对方未进来就开始跑图/刷资源”的体验问题。

## AI 辅助记录（按作业要求可提交）

### AI 使用方式说明
在实现联机玩法与同步一致性（尤其是：死亡锁定、掉落同步、胜负 UI 一致性、启动 HUD 流程、打包退出行为）时使用 AI 进行辅助：
- 让 AI 提供 **排查思路**、**实现方案**、以及**可直接落地的代码修改建议**
- 由我在本地应用修改、编译并验证最终效果

### AI 辅助任务清单（已按当前项目实现填写）

#### 任务 A：玩家死亡后禁止交互（移动/技能/拾取/投递/碰撞）
- **目标**：玩家死亡后不再参与战斗与资源交互；敌人不再攻击尸体；避免死亡玩家继续触发拾取/投递。
- **实现要点**：
  - 玩家死亡后统一禁用 `PlayerMovement / PlayerAttack / PlayerSkills / PlayerGoldCarrier`
  - 关闭 `Collider2D` 与 `Rigidbody2D` 物理模拟
  - 敌人索敌与伤害判定跳过 `PlayerHealth.IsDead`

#### 任务 B：携带金块的野怪死亡掉落在 Client 不显示/不可拾取
- **目标**：Host/Client 双方都能看到掉落金块且可拾取。
- **实现要点**：
  - 统一让掉落金块由 **服务器网络生成并 Spawn**
  - 对“携带状态”与“父子层级”不同步造成的不可拾取做兜底判断
  - 优化 Client 侧野怪翻面抖动（减少左右抽搐观感）

#### 任务 C：胜负 UI 在 Client/Host 显示不一致（Victory vs Game Over）
- **目标**：多人胜利/失败时两端显示一致。
- **实现要点**：
  - 调整 `SyncVar` 字段顺序，让胜负结果先写入，再触发 `_matchEnded` hook 触发 UI。

#### 任务 D：玩家携带金块死亡时掉落全部金块
- **目标**：玩家死亡时掉落其携带的全部金块，双方可见可拾取。
- **实现要点**：
  - 服务器在玩家死亡时读取 `carryCount`，一次性生成 `GoldPickup.amount = carryCount` 的地面金块并 `NetworkServer.Spawn`
  - 清空玩家背包 `carryCount = 0`

#### 任务 E：启动 HUD 改为“单人开始/双人开始”，并支持地址/端口编辑
- **目标**：替换原 Mirror HUD 按钮，提供可演示的单人/双人启动流程。
- **实现要点**：
  - 单人：直接 Host 开局
  - 双人：优先尝试作为 Client 连接；失败则自动 Host 并等待对方加入
  - 等待期间锁定本地玩家操作，对方加入后解除

#### 任务 F：打包版 Quit 退出卡死/无效
- **目标**：打包后点击 Quit 能可靠关闭窗口。
- **实现要点**：
  - 避免在退出时执行可能阻塞主线程的“优雅断开网络”
  - 直接 `Application.Quit()`，并在 Standalone 下延迟 `Environment.Exit(0)` 兜底
